# boot-selector-switch

A physical rotary switch on my PC case that picks which OS to boot. Turn the knob, reboot, done.

## How it works

A 6-position rotary switch is wired to a Raspberry Pi Pico 2, which shows up as a USB HID device. It has a single HID report that is a byte that represents the position of the switch. A UEFI application (the "efi-shim") runs before systemd-boot, reads the switch position over USB, sets the `LoaderEntryOneShot` EFI variable to the right boot entry, and then chain-loads systemd-boot. systemd-boot sees the variable and boots the selected OS.

The switch position to entry mapping is configured via a simple config file on the ESP, generated by the NixOS module.

## Parts

- **boot-selector-switch-efi-shim** - The UEFI app. Reads the USB device, reads the config, sets the EFI variable, chain-loads systemd-boot.
- **pico-firmware** - Embassy-rs firmware for the Pico 2. Reads 6 GPIO pins (one per switch position) and reports the active position over USB HID.
- **virtual-switch** - USB/IP emulator of the switch for development without hardware. Has a little TUI to pick a position.
- **test-efi-stub** - Fake EFI binaries that just print a name and shut down. Used for the Windows entry in the test VM.
- **nixos-module** - NixOS module that installs everything, generates the config file, creates a `nixos-latest.conf` entry, and registers the efi-shim in the UEFI boot order.

## Building

```bash
nix build .#efi-shim             # production UEFI binary
nix build .#efi-shim-qemu        # UEFI binary with qemu compat
nix build .#pico-firmware         # ARM ELF for the Pico 2
nix build .#virtual-switch        # USB/IP emulator

nix run .#test-vm                 # boot a NixOS VM (no USB device)
nix run .#test-vm-usb             # boot a NixOS VM with USB passthrough (needs sudo)
nix run .#flash-pico-firmware     # flash the Pico over SWD
```

## Test VM

There's a NixOS VM for testing the whole boot chain without real hardware. It has systemd-boot, the efi-shim, a fake Windows entry, and netboot.xyz.

```bash
# Without USB hardware (efi-shim runs but can't find the switch, falls through to default boot)
nix run .#test-vm

# With the real Pico plugged in (USB passthrough, needs sudo)
nix run .#test-vm-usb
```

The disk image is rebuilt fresh every run so you always get the latest code. EFI variables persist across reboots in `.vm-state/` so debug mode sticks.

You can toggle debug mode from inside the VM with `boot-selector-debug on` and then `reboot` to see the verbose output.

## NixOS module

```nix
{
  inputs.boot-selector-switch.url = "github:giodamelio/boot-selector-switch";

  outputs = { self, nixpkgs, boot-selector-switch, ... }: {
    nixosConfigurations.myhost = nixpkgs.lib.nixosSystem {
      modules = [
        boot-selector-switch.nixosModules.default
        {
          boot.loader.boot-selector-switch = {
            enable = true;
            positionMap = {
              "1" = "nixos-latest.conf";
              "2" = "windows.conf";
              "3" = "netbootxyz.conf";
            };
          };
        }
      ];
    };
  };
}
```

The module handles installing the EFI binary, generating the config file, creating the `nixos-latest.conf` boot entry, and registering the shim first in the UEFI boot order.

There's also a `boot-selector-debug` helper script that gets added to the system. It toggles a debug mode EFI variable that makes the shim show extra info and wait for a keypress on the next boot.
